# portal-cnweb-hub

本仓库用于归档和发布多个独立 HTML 页面。

## 项目简介

这是一个通用型 HTML 页面归档仓库，收集并托管各类独立网页资源。仓库内容不针对任何特定域名或网站，旨在提供一个结构化的页面存储与分发空间。

## 目录说明

```
portal-cnweb-hub/
├── pages/          # 存放主要 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── archive/        # 历史版本或已归档的页面
└── README.md       # 本文件
```

- `pages/`：当前可访问的 HTML 页面集合。
- `assets/`：页面所需的公共资源文件，按需组织子目录。
- `archive/`：不再更新或已过时的页面，保留供参考。

## 页面归档说明

- 每个 HTML 页面独立存放在 `pages/` 目录下，命名应反映页面内容或功能。
- 页面引用的资源应尽量使用相对路径，确保在不同部署环境下正常工作。
- 如需添加新页面，请放入 `pages/` 并更新本文档的页面列表（可选）。
- 归档页面移至 `archive/`，并在文件名后添加日期后缀以示区别。

## 维护说明

- 仓库维护者负责审核新增页面，确保内容合规且不包含恶意代码。
- 页面更新时，旧版本可移入 `archive/` 或直接覆盖（建议保留历史版本）。
- 不鼓励存放动态内容或依赖外部服务的页面，以免因外部变更导致失效。
- 提交 PR 或 Issue 时，请清晰描述页面用途及变更内容。

## 使用方式

可通过 GitHub Pages 或其他静态托管服务直接部署本仓库内容。部署后，`pages/` 目录下的 HTML 文件可通过对应 URL 直接访问。
